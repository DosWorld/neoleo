12.4 Running shell commands

M-x shell

This command runs the shell command you specify, reads in each line output to stdout by the command, and executes them as if you had typed them in to execute-command. It is similar to read-commands.

The features/non-features of shell are:

  • The command must be somewhere that neoleo can find it: typically in your PATH environment variable.
  • It is language-neutral. You can write in any language: bash, python, perl, a binary, etc..
  • Two-way communication between neoleo and the command is not possible. neoleo just takes commands that are printed to stdout and executes them. Unlike read-commands, though, the commands can be generated programmatically.
  • neoleo does not automatically update the display after the execution of each command. You will therefore have to wait until the script is finished to see the effect.

Example 1

Here is an example, which is a copy of the bash script oleo42.sh in the examples directory: 

#!/usr/bin/env bash
# Example script that enters the value 42 in the next cell down
# Ensure that the script is somewhere in your $PATH
# Run it using
#       M-x shell oleo42.sh


#echo `date` hello from oleo42.sh >> /tmp/oleo-shell
echo down-cell
echo edit-cell 42

This will move the cursor down one cell, and enter the value 42.

To execute the script, ensure that it is somewhere in your PATH, and type: 

M-x shell <RET> oleo42.sh <RET>

A more elaborate script, written in python, is examples/oleo-brownian.py.

Example 2: Generate random numbers

The following script examples/rnd5) generates 5 random numbers and puts them in your spreadsheet columnwise:

echo i
